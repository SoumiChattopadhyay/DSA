int n = 5;
char c = (char)(n - '0');
System.out.println(c);


'0' in ASCII = 48
n - '0' = 5 - 48 = -43
(char)(-43) → this wraps around in Java’s unsigned 16-bit char type.

Java interprets this as:
(char)(-43) = (char)(65536 - 43) = (char)65493, which is a strange non-printable or weird Unicode character and might show nothing,a square box (�),or some random symbol depending on your console.


Java’s char type is:
Unsigned (no negative values)
16-bit (can store values from 0 to 65535)

But when you cast a negative number (like -43) to char, Java can’t store negative numbers, so it wraps around using modular arithmetic — like a clock looping back.

🧮 How it wraps:
(char)(-43) == (char)(65536 + (-43)) == (char)(65493)
