🤍🌺Annotation(Ex: @Override):
    It is internally an interface.

🤍🌺Default method:
    It is possible to give default implementation to interfaces.
    The primary motivation was to expand the interfaces without breaking the existing code.
    Suppose u add a method in interface then it needs to implemented by all of the classes implementing the interface.
    The class's method implementation has more priority than the default implementation(interface default method)
    Duplicate default methods from 2 or more interfaces cannot be inherited by a class or another interface 

🤍🌺Static method:
    Static methods never get inherited.
    Inheritance means possibility of overriding
    overriding means it depends on objects coz its runtime polymorphism
    which method runs depends on the actual object
    But wait static methods are never accessed through objects they are totally not dependent on objects
    So they cannot be overridden
🤍🌺So if there are any static methods in interfaces they must contain their function body also. 
They would be used from the interface only i.e. they can be accessed through the interface name
And they won't be inherited by any other interface or class

The access modifiers for the overridden methods should be same or better
Like if the method getting overridden(the parent class's method) is protected then the method overriding it must be public or protected. It cannot be more restrictive than protected, it cannot be private or default.
